## Project Overview

**after-six** (加班统计) is a browser extension that automatically calculates and displays overtime hours on the EHR attendance calendar at `ehr.supcon.com`. It injects a summary and per-day overtime badges into the calendar view, replacing the manual developer-tools script workflow described in `想法.md`.

## Commands

- `pnpm dev` — Start dev server (Chrome). Use `pnpm dev:firefox` for Firefox.
- `pnpm build` — Production build. Output goes to `.output/`.
- `pnpm zip` — Build and package as `.zip` for store submission.
- `pnpm test` — Run all tests with Vitest (`vitest run`).
- `pnpm test -- --watch` — Watch mode for tests.
- `pnpm test -- utils/__tests__/calculator.test.ts` — Run a single test file.

## Architecture

### WXT Framework

Built on [WXT](https://wxt.dev) which provides `defineBackground()`, `defineContentScript()`, and browser API globals (`browser.*`) without explicit imports. TypeScript config extends `.wxt/tsconfig.json` (generated by `wxt prepare` / `postinstall`).

### Entrypoints (`entrypoints/`)

- **`background.ts`** — Service worker. Handles toolbar icon click (opens/focuses the EHR tab) and processes `getOvertimeStats` messages from the content script. Fetches daily attendance data in batches of 5 using `Promise.allSettled`, then runs overtime calculation.
- **`ehr.content.ts`** — Content script injected into `ehr.supcon.com`. Locates the attendance calendar inside an iframe (`#portalframe`), extracts `staff_id` from a hidden input, and communicates with the background script. Renders:
  - A summary line (total overtime for the month)
  - Per-day badges on calendar cells (`td.fc-day`)
  - Automatically re-fetches when the user navigates months (MutationObserver + click/change event delegation with 800ms debounce)

### Utilities (`utils/`)

- **`api.ts`** — `fetchDailyAttendance(staffId, date)` — POST to the EHR API for a single day's attendance records.
- **`calculator.ts`** — `calculateOvertime(records)` — Core business logic. Calculates workday vs weekend overtime based on clock-in/out times and `datetypename` (工作日/休息日/法定节日). Workday overtime start time is interpolated between 17:30-18:30 based on clock-in time (before 8:00 → 17:30, after 9:00 → 18:30, in between → linear). Also exports `getDaysArray()` and `timeToString()`.
- **`storage.ts`** — `getStaffId()` — Reads `staff_id` from `localStorage.personInfo` (currently unused by the active flow, which reads from iframe DOM instead).

### Types (`types/index.ts`)

All shared interfaces: `AttendanceRecord`, `ApiResponse`, `OvertimeDetail`, `OvertimeStats`, `MessageRequest`, `MessageResponse`.

### Tests (`utils/__tests__/`)

Vitest with `happy-dom` environment. Tests cover the calculator logic, API call construction, and storage utility.

## Key Domain Knowledge

- The EHR system returns `sb_dk_time` (上班打卡) and `xb_dk_time` (下班打卡) with fallbacks (`_time2`, `_time3` for multiple punch records).
- `datetypename` values: `工作日`, `休息日`, `法定节日`, and others (调休/年假 etc.).
- Overtime is measured in milliseconds throughout the codebase; display converts to hours+minutes.
- Content script uses CSS selector + XPath fallback to locate the calendar container, since the EHR DOM structure is fragile.

## CI/CD

GitHub Actions (`.github/workflows/release.yml`): push a `v*` tag → `pnpm install` → `pnpm zip` → upload `.zip` to GitHub Release. Uses Node 22 and pnpm 9.15.4 (pinned via `packageManager` field).
